Automating Board Game Geek

Introduction

In this lab exercise, you’ll be writing a test plan and automated tests for boardgamegeek.com.

Instructions

Outline

  1. Explore the website for about 20 minutes to identify some user flows.

  2. Plan at least 5 different tests.

  3. Write your tests!

1. Explore

  • Figure out what the purpose of the website is and what users would come here to do

  • Take notes as you explore and include them in your GitHub repo

  • Look out for things that are broken

2. Plan

  • Make sure you plan for at least 5 automation tests

  • Try to find varied things to test so that you’re not doing the same thing over and over

  • Include your test plan in your repo

3. Test

  • There are no materials to download for this lab so you’ll be starting from scratch!

  • Create a folder and navigate to it in your command line

  • Run npm init -y to create a package.json boiler plate

  • Use npm to install @types/jest, @types/selenium-webdriver, chromedriver, jest, and selenium-webdriver

  • Create your test file

  • Set up your file

    • bring in Builder and Capabilities from selenium-webdriver

    • bring in chromedriver

    • create your driver

    • write a beforeAll and afterAll

  • Write your tests

  • If you finish 5 and still have time, keep going!

When you’re done, make sure to push everything to GitHub
